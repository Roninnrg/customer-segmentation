# Сегментация клиентов магазина (K-Means)

Мой второйпроект, где я разбирался с обучением без учителя (Unsupervised Learning). Задача — взять сырую базу покупателей торгового центра и автоматически разделить их на группы по поведению, чтобы маркетологи могли делать им нормальные точечные предложения, а не спамить всех подряд одинаковыми письмами.

## Что внутри проекта:
* **Анализ данных:** Исследовал базу клиентов — их возраст, пол, годовой доход и индекс трат, который им присвоил магазин.
* **Метод локтя (Elbow Method):** Написал цикл, чтобы посчитать инерцию (WCSS) для разного количества групп от 1 до 10. Построил график, на котором четко виден изгиб («локоть») на цифре 5. Это доказало, что базу нужно делить ровно на 5 сегментов.
* **Кластеризация:** Обучил финальную модель `KMeans` с параметром `init='k-means++'`, чтобы начальные точки не вставали криво и результат всегда был стабильным.
* **Визуализация:** Построил итоговый Scatter Plot, где каждая группа подсвечена своим цветом, а черными крестиками отметил центроиды (математические центры групп).
* **Сохранение модели:** С помощью `joblib` упаковал готовую модель в файл `customer_kmeans_model.pkl`. Теперь, если придет новый клиент, модель сразу определит его в нужную группу без переобучения.

## Какие группы клиентов получились:
1. **«Середняки»:** Средний доход, средние траты. Самая большая и стабильная группа.
2. **«Рассудительные богачи»:** Зарабатывают много, но тратят мало. Отличная цель для маркетологов, чтобы предложить им что-то премиальное.
3. **«Золотая жила (VIP)»:** Высокий доход и высокие траты. Самые ценные клиенты, которых надо удерживать любыми способами.
4. **«Транжиры»:** Доход низкий, но тратят очень много (импульсивные покупки).
5. **«Экономные»:** Низкий доход, низкие траты. Люди с жестким бюджетом, которых интересуют только скидки.
